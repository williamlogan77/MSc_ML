Variables that are not numerical will need to be converted into numeric values if they are to be used. This can be done in a variety of ways, and is done later on in this notebook manually - though this can be done using a method from SKlearn.

To approach the problem of needing only numerical values, a lot of the variables are missing a fair amount of values and encoding all of the categorical variables into numerical values may be a waste of time/computing power. Therefore, the variables to encode are decided upon before encoding them. The first step in this approach is to remove any variable with more than 40% of its value that are NaN (or any synonymous value).

In [ ]:
# Create a list that contains a boolean that defines whether the variables has more than 40% of its variables as NaN (or a substitute thereof).
cond = (data.isna().sum() / data.shape[0] < 0.4).values
# Apply previous list to the dataframe, removing any variables that have the value `False`
data = data.loc[:, cond]

The next choice to make is more a matter of common sense. There are some unique variables like: image_url, description, region_url, url, VIN and id. Since these are unique to the individual posting, these can also be removed as they do not give any information and without looking into further do not hold any predictive power (from face value at least). Furthermore, some variables are subjective, for example title_status which describes the current status of the car. This may contain some information regarding the price, however it is removed as there are more important variables to investigate. posting_date is of a similar informative level and thus is removed.

In [ ]:
# Remove aforementioned columns (specifying inplace to remove them from the original data)
data.drop(columns=["image_url", "description", "region_url",
                   "url", "id", "title_status", "VIN", "posting_date"], inplace=True)

Another point of concern is the amount of unique values in non-numerical values can be challenging to deal with - especially if they all have to be encoded. Any non-numerical value with less than 10 unique values can be dealt with fairly easily and thus, any variable with more than 10 unique values is dropped.

As mentioned, there are still some values that are NaN that need to be dealt with. This can be done using a method from SKlearn to fill in the values with whatever necessary. In this case, the median value from each variable is used to replace these values. The reason the median is used, is due to the fact that the median value chosen is a known value from the data set, whereas something like the mean may be a decimal value that does not fall into one of the encoded values.

Model Fitting¶

There are several directions to take the model fitting. In this report, I will fit the variables to two separate models; a decision tree regressor model and a random forest model. This will help to establish a "better" model and it will also help to provide some sort of comparison/benchmark to help decide how good the model is.

I will assess the quality of fit of each model and compare them side by side and then perform cross validation on the data (more on this later in the fitting).

The first thing to do is create a training and testing set of data into an 80/20 split respectively. This is so that fit of the model can be assessed on data that it has not seen before.

In [ ]:

To take the analysis of each model even further, the cross validation score (CVS) can be calculated. The CVS is calculated by doing a similar thing as done above. Cross validation splits the data into different sets of data (in this case 10 different sets) and trains the data and compares it to the testing data. Here it is greatly simplified by SKlearn's algorithm and instead of performing the split manually (with the function train_test_split) it is done automatically and semi-randomly.

The main reason for CVS here is to identify that the models are formed independently to the splitting of the data.

In [ ]:
# Import the cross validation method 
from sklearn.model_selection import cross_val_score

# Choose the number of folds (the number of times the data is split)
K = 10

# For cross-validation a new model needs to be initialised
new_RF = RandomForestRegressor(n_estimators=100, random_state=2607, n_jobs=-1)
new_DT = DecisionTreeRegressor(random_state=2607)


# Perform cross-validation, with all the data fed to it
DT_scores = cross_val_score(new_DT,
                            data.filter(
                                ["transmission", "drive", "year", "odometer"]),
                            data.filter(["price"]), cv=K,
                            n_jobs=-1, scoring="neg_mean_squared_error")


RF_scores = cross_val_score(new_RF,
                            data.filter(
                                ["transmission", "drive", "year", "odometer"]),
                            data.filter(["price"]), cv=K,
                            n_jobs=-1, scoring="neg_mean_squared_error")

# Since the scoring method was negative mean squared error, the actual error is the square of the positive value
DT_scores = np.sqrt(-DT_scores)
RF_scores = np.sqrt(-RF_scores)

Now calculate the relevant metrics for the cross-validation process. Here, the lower the score, the better the model.
